Description:
Abstract:
Green infrastructure increasingly is used to ameliorate water quality and quantity problems caused by runoff in cities. Studies show how the spatial distribution of these Green Stormwater Infrastructure (GSI) sites are unevenly distributed relative to socioeconomic and demographic groups. Often this is described as an indicator of perpetuated environment injustice, given the purported social and environmental benefits of GSI. To assess equity, researchers often examine either who gets what with respect to environmental ‘goods’ such as tree canopy and other green infrastructures, or investigate the procedures, decision making processes, and power structures pertaining to planning processes.

The data publication contains GSI location data collected from 2013-2021. It also contains information regarding who lives in nearby New Haven, Connecticut. Specifically, these data include four (4) vector polygon shapefiles representing census block groups, neighborhoods, sewer, and watershed outlines and two (2) point shapefiles representing green infrastructure and combined sewer overflow locations. An R Markdown (.rmd) file is also included containing analysis steps.
Purpose:
The intention of providing these data is to accelerate and enable other future researchers interested in GSI and Environmental Justice, and to allow students to access data for use in classes in geographic information systems, urban studies, statistics, environmental justice, and related.

Attribute_Accuracy:
Attribute_Accuracy_Report:
Because of New Haven’s relatively small geographic and population size (relative to other often-studied areas), and the few green stormwater infrastructure (GSI) actors on the scene, we can be reasonably confident that the entire universe of street-side bioswales are accounted for.

Census has its margins of error, the land cover is documented elswhere, the GSI points are accurate enough to land in the correct neighobrhood or block group via a spatial join. As described in Locke et al. (2021), we manually reassigned three points from an industrial area (Long Warf) to the adjacent neighborhood.

Locke, Dexter H.; Phillips de Lucas, Amanda K.; Murphy-Dunning, Colleen; Henning, Dawn; Zinn, Giovanni. 2021. Contextual considerations of green stormwater infrastructure siting. Cities and the Environment 4(2): 2. https://doi.org/10.15365/cate.2021.140202

Process_Step:
Process_Description:
The City of New Haven’s Engineering Department uses GIS as a data management tool for its Green Stormwater Infrastructure installations. The GSI geodatabase was created using three different methods. For the smaller projects (less than 10 GSI), points were created using ArcMap/ArcGIS Pro desktop version. They created points with attributes that were removed to create "gi_cartographic.shp".

Process_Step:
Process_Description:
The shapefile "neighborhoods.shp" boundaries and demographic data came from DataHaven. DataHaven is a non-profit organization based in New Haven that developed neighborhood boundaries with the city in 2012, in order to support the Comprehensive Plan Data Book. Most of the neighborhood boundaries align with 2010 Census tract boundaries, but there are a few instances where Census tracts or block groups are split, to better match locally-relevant understandings of neighborhoods. DataHaven created neighborhood estimates by allocating Census data to corresponding neighborhoods using the share of population or households that fell within each geographic area. The very small number of people and households living within the boundaries of the city’s Long Wharf neighborhood, which is predominantly a commercial and industrial area, were automatically assigned to the directly adjacent neighborhood called the Hill.

Entity_and_Attribute_Overview:
Below you will find a list and description of the files included in this data publication.


DESCRIPTION FILES (2)

1. \Data\census_block_groups_field_descriptions.csv: Comma-separated values (CSV) file containing the attributes and attribute definitions of \Data\census_block_groups.shp.

2. \Data\neighborhoods_field_descriptions.csv: Comma-separated values (CSV) file containing the attributes and attribute definitions of \Data\neighborhoods.shp.

Both of these CSV files contain the following:
column number = Column number
name = Name of variable
alias = Variable name, information, or alias from source
definition = Description of variable (or blank if defined in alias)
source = Source of variable (Census = Census, DataHaven = DataHaven, internal = assigned internally by author, original = original - as provided/obtained, derived from ... = data derived high-resolution land cover from South Central Regional Council of Governments (SCRCOG) and the University of Vermont (UVM) the Spatial Analysis Lab (The SAL))


DATA FILES (7)

1. \Data\All_Spatial_Data.gpkg: OGC GeoPackage (GPKG) containing all spatial data files (2-7) included in this publication.

2. \Data\census_block_groups.shp: Shapefile (and associated files) containing a vector polygon representation of census block groupings from the U.S. Census within New Haven, Connecticut. A complete description of the attributes in this file are available in \Data\census_block_groups_field_descriptions.csv.

3. \Data\neighborhoods.shp: Shapefile (and associated files) containing a vector polygon representation of neighborhood boundaries in New Haven, Connecticut. A complete description of the attributes in this file are available in \Data\neighborhoods_field_descriptions.csv. (Note: ESRI modifies the variable names, see the *.Rmd file which renames the columns on import.)

For the following data files (4-7), only standard attributes, generated by Esri, are found in each shapefile:
FID = Numeric identifier automatically generated by Esri
Shape = Feature geometry automatically generated by Esri
FID_1 = Numeric identifier automatically generated by Esri

4. \Data\CSOs.shp: Shapefile (and associated files) containing a vector point representation of combined sewer overflow locations.

5. \Data\dt_shed.shp: Shapefile (and associated files) containing a vector polygon representation of the downtown watershed.

6. \Data\gi_catographic.shp: Shapefile (and associated files) containing a vector point representation of green infrastructure locations.

7. \Data\shed_outlines.shp: Shapefile (and associated files) containing a vector polygon representation of sewer watershed outlines.


SUPPLEMENTAL FILES (1)

1. \Supplements\New_Haven_GI_CATE.Rmd: R Markdown (RMD) file containing analysis steps used for the original publication.
